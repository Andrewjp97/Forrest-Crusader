The Making

Follow Along As We Develop Forrest Crusader to Make Your Own Version

Before beggining, you should be comfortable with Mac OS X, iOS, XCode, and the Swift programming language. If you feel at all uncomfortable with any of these things, please use available resources like the tutorials at RayWenderlich.com to gain familiarity, and then come back. You will also need to be a registered apple developer enrolled in the iOS developer program. Visit Developer.Apple.com for more information.

Let's start by opening XCode and creating a new project. Create a new SpriteKit Game, use the Swift Language, and give it a name. Then click create.



After you have created your project, take a look around at some of the files, and then open AppDelegate.swift. Take a look at some of the code. Take note of the first method, as you will need to customize the application and pull data from storage in this method later.

Delete GameScene.sks, we won't be using it in this game. Open up GameScene.swift and replace the contents with the code on the right. What we have done here is eliminate the need for a scene .sks file (SpriteKit Serialized Data file) and initialize our new scene directly in code. We pass in the size of our viewcontroller's view's frame to the scene, and then present that scene. We are also displaying some diagnostic information such as the node count and draw rate on the screen. When we prepare to release our app we will remove those lines of code.

There is still something we need to do before getting too far ahead of ourselves, and that is set the device orientations. Check landscape left and right for both iPad and iPhone.

Next we need to add the JSTileMap helper class, which will transform our Tiled maps for each world into sprites that we can present on screen. We need to first create on objective-c bridging header, which is best done by creating an objective-c implementation file and letting XCode create one for us. Then delete the file, and import JSTileMap.h and .m, as well as LFCGzipUtility.h and .m. Then add an import statement to the bridging header and link the SpriteKit, UIKit, Foundation, and libz.dylib binaries to our project. If you need the JSTileMap and LFCGZipUtility files, you can download them for free on GitHub.
